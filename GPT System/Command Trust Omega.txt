# FS25_Modding_OS_v1.0

## 🔥 PURPOSE

This file defines the core operating system layer for FS25 modding development.

It introduces:

- Natural language command interpretation
- Dataset revalidation system (Omega)
- Trust layers for mod references
- File + log visibility awareness
- UI + Lua + XML diagnostic routing
- Persistent mod knowledge architecture

---

# 🧠 SECTION 1 — CORE DESIGN PHILOSOPHY

## PRINCIPLES

- Treat all data as non-trustworthy until revalidated
- Never assume file availability from upload alone
- Prefer explicit retrieval over memory assumptions
- Convert natural language into structured commands
- Resolve ambiguity through layered interpretation

---

# 🧩 SECTION 2 — COMMAND INTERPRETER (OMEGA LAYER)

## 🧠 CORE IDEA

All user input is interpreted as either:

- COMMAND
- INTENT
- DATASET QUERY

---

## 🔥 PRIMARY COMMANDS

### 🧩 REVALIDATE

```text
REVALIDATE <dataset>

Triggers full Omega validation pipeline.

🧠 BOOT
BOOT

Initializes system state, loads known-good references, and runs dataset health check.

🔍 VERIFY
VERIFY <file/dataset>

Lightweight version of REVALIDATE (single-pass search only).

🔁 REFRESH
REFRESH <dataset>

Re-attempts retrieval without full Omega expansion.

📊 STATUS
STATUS <dataset>

Returns last known trust state without re-search.
