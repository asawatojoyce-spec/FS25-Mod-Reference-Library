# PURPOSE

This document extracts **repeatable UI success patterns** from known-good FS25 mods and base-game systems.

- where correct injection happens

- why certain tab systems succeed
- how GIANTS engine expects UI registration to occur

Not theory.

Behavior.

---

# 🧩 VERIFIED WORKING UI PATTERN (BASE MODEL)

All known-good mods follow this structure:

Script loaded early (modDesc <extraSourceFiles>)
UI class defined BEFORE hook execution
InGameMenu.onLoad intercepted via Utils.appendedFunction
UI frame created DURING menu construction
XML loaded via g_gui:loadControlFromFile
Page registered via registerPage
pagingTabList rebuilt immediately


---

# 🔥 CRITICAL SUCCESS FACTOR

## ✔ Timing alignment is everything

Working mods do NOT:

- wait for loadMapFinished
- delay UI injection
- register pages after UI freeze

They inject:

> DURING InGameMenu construction window

---

# 🔍 COMMON SUCCESS PATTERNS (EXTRACTED)

## 1. EARLY HOOK REGISTRATION

```lua
InGameMenu.onLoad =
    Utils.appendedFunction(InGameMenu.onLoad, MyMod.onMenuLoad)
``` id="p1q9a2"

✔ Always happens BEFORE menu finalization

---

## 2. IMMEDIATE PAGE CREATION

```lua
local page = MyUIClass.new(inGameMenu)
inGameMenu:registerPage(page, index, icons)
``` id="xk2m8q"

✔ Page created inside menu lifecycle window

---

## 3. CONTROL TREE BINDING

```lua
g_gui:loadControlFromFile(xmlPath, profile, page, menu)
``` id="t9z4lm"

✔ XML is attached as SUBFRAME, not full screen GUI

---

## 4. IMMEDIATE CACHE REBUILD

```lua
pagingTabList:rebuildTabs()
``` id="r7v3cd"

✔ Forces UI refresh before freeze point
