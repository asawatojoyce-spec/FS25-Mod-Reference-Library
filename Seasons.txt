## FS25 UI CRITICAL RULES

For UI / InGameMenu issues, always check:

- InGameMenu.onLoad timing
- registerPage timing
- pagingTabList rebuild timing
- XML binding correctness
- loadControlFromFile usage
- TabbedMenuFrameElement inheritance
- modDesc script load order

Primary UI rule:

UI injection must occur during InGameMenu.onLoad.

Unsafe timing:

- loadMapFinished
- post-render hooks
- late initialization after pagingTabList cache lock

Known-good pattern:

1. Script loaded early through modDesc extraSourceFiles.
2. UI class defined before hook execution.
3. InGameMenu.onLoad intercepted.
4. Frame created during menu construction.
5. XML loaded with g_gui:loadControlFromFile.
6. Page registered with registerPage.
7. pagingTabList rebuilt immediately if required.

## SEASONSANIMALS ARCHITECTURE RULES

Core philosophy:

- lightweight simulation system
- modular
- transparent
- XML-driven
- removable without destroying saves

Core layers:

CORE SIMULATION
DAILY PROCESSOR
PLACEABLE XML / SAVEGAME XML
UI SNAPSHOT READER

Critical rule:

UI must never drive simulation.
UI must only read XML/state snapshots.

Update cycle:

- Hourly updates belong to the base game engine.
- Daily updates belong to SeasonsAnimals recalculation.
- Persistent state belongs in XML.

Savegame file:

savegame#/seasonsAnimals.xml

Removing the mod should not destroy vanilla animals.
Reinstalling the mod should restore SeasonsAnimals data.
